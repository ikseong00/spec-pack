# Projection Matrix

이 문서는 planning record block이 어느 core doc section으로 projection되어야 하는지 고정한다.

## Core Mapping

- `domain_shape_snapshot`
  - `PROJECT-THESIS.md`: product shape and operator-visible constraint summary
  - `PRD.md`: scope-shaping modifiers and wedge constraints
  - `UX-IA.md`: actor, surface, and loop constraints
- `user_model`
  - `PROJECT-THESIS.md`: target user
  - `PRD.md`: representative use cases
  - `UX-IA.md`: actor and context constraints
- `recent_examples`
  - `PROJECT-THESIS.md`: anchor example
  - `RESEARCH.md`: evidence summary
- `use_case_cards`
  - `PRD.md`: primary use case
  - `UX-IA.md`: flow basis
- `workflow_contract`
  - `UX-IA.md`: lifecycle, queue, and handoff map
  - `SCREEN-SPECS.md`: state-bearing surfaces
  - `EXECUTION-HANDOFF.md`: workflow slice scaffolding
- `market_evidence_brief`, `competitor_table`
  - `RESEARCH.md`
- `hypothesis_register`
  - `PLANNING-RECORD.md`
  - `EXECUTION-HANDOFF.md` when next step is experiment
- `business_viability_snapshot`, `commercial_model`, `metric_hierarchy`
  - `BUSINESS-OPS.md`
- `screen_specs`
  - `SCREEN-SPECS.md`
- `execution_handoff`
  - `EXECUTION-HANDOFF.md`

## Derived Artifact Rule

- `One-Page Core Artifact`는 derived-only다
- canonical source of truth는 항상 `PLANNING-RECORD.md`와 해당 owning doc다
- one-page slot 값은 이 문서의 `Fixed Slot Ownership`에 있는 owning doc를 override할 수 없다
- one-page 수정이 owning doc와 충돌하면 one-page를 stale로 보고, owning doc를 기준으로 다시 생성한다
