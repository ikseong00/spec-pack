# PROJECT-THESIS

- one_line_product: `규제성 클리닉 초진/재진 intake와 예약 확정, 재방문 recovery를 한 화면 흐름으로 묶는 환자용 웹 intake`
- primary_target: `추가 검사나 후속 진료가 필요한 외래 환자`
- primary_actor: `patient`
- secondary_actor: `clinic intake nurse`, `care coordination desk`
- why_this_segment_first: `예약 hold와 재방문 누락이 운영 리스크와 환자 불안을 동시에 키운다`
- first_value_moment: `환자가 제출 직후 현재 예약 상태와 다음 행동을 확실히 본다`
- must_not_hide_states: `hold`, `confirmed`, `needs callback`, `revisit needed`
- success_condition: `환자가 전화 없이 예약 상태와 재방문 recovery path를 이해한다`
